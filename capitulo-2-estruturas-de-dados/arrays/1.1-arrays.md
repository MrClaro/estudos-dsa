## Introdução

As **Arrays**, ou vetores, são a estrutura de dados linear mais fundamental. Elas consistem em uma coleção de elementos do mesmo tipo, armazenados em posições **contíguas** (vizinhas) na memória RAM.

A grande vantagem das arrays é que o acesso a qualquer elemento é feito em tempo constante **$O(1)$**. Como vimos na parte de RAM, isso acontece porque o computador calcula o endereço exato do item instantaneamente usando o índice.

**Analogia:** Pense em uma prateleira de livros onde cada espaço tem um número gravado. Se você sabe que quer o livro na posição 5, você não precisa olhar os livros 1, 2, 3 e 4; você move sua mão diretamente para o espaço 5.

## Operações e Complexidade

As operações em arrays variam drasticamente de custo dependendo de **onde** ocorrem:

- **Acesso O(1):** É a operação mais rápida. Basta usar a sintaxe `array[índice]`. O tempo é o mesmo, não importa se o array tem 10 ou 10 milhões de itens.

- **Busca:**
  - **Busca Linear O(n):** Se o array está desordenado, você precisa olhar item por item. No pior caso, o item é o último ou não existe.
  - **Busca Binária O(\log n):** Se o array estiver **ordenado**, você pode usar a técnica de "dividir para conquistar/divide and conquer", cortando o espaço de busca pela metade a cada passo.

- **Inserção:**
  - **No Final O(1):** Se houver espaço disponível, basta colocar o item na próxima posição.
  - **No Início ou Meio O(n):** Esta é a parte "cara". Para inserir alguém no índice 0, você precisa "empurrar" todos os outros elementos uma posição para a direita para abrir espaço.

- **Remoção:**
  - **No Final O(1):** Basta ignorar ou apagar o último elemento.
  - **No Início ou Meio O(n):** Assim como na inserção, você precisa deslocar todos os elementos à direita para a esquerda para não deixar "buracos" na memória contígua.

## Arrays Estáticos vs. Dinâmicos

É importante diferenciar como as linguagens lidam com o tamanho das arrays:

1. **Arrays Estáticos:** Têm tamanho fixo definido na criação (comum em Java puro: `int[] arr = new int[5]`). Se você precisar de mais espaço, terá que criar um novo array maior e copiar os dados.
2. **Arrays Dinâmicos:** Podem crescer conforme o necessário (como o `ArrayList` em Java ou o `Array` no JavaScript/TypeScript). Por baixo dos panos, eles continuam sendo arrays contíguos, mas a linguagem gerencia a criação de novos espaços e a cópia de dados para você quando o limite é atingido.

## Arrays Homogêneos vs. Heterogêneos

- **Homogêneos:** Todos os elementos são do mesmo tipo (ex: `int[]` em Java). Isso permite otimizações de memória e acesso.
- **Heterogêneos:** Podem conter tipos diferentes (ex: `Object[]` em Java ou `Array` em JavaScript). Isso oferece flexibilidade, mas pode ter um custo de desempenho e uso de memória, pois o sistema precisa armazenar informações adicionais sobre o tipo de cada elemento.
